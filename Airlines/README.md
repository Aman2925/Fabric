# 📌 Project Overview

Airline operations involve millions of daily transactions across multiple systems including flight scheduling, passenger bookings, aircraft maintenance, airport operations and weather monitoring. These systems often operate independently, making it difficult to generate reliable business insights without a centralized analytics platform.

This project demonstrates how **Microsoft Fabric** can be used to design and implement a complete **end-to-end data engineering and business intelligence solution** using the **Medallion Architecture**.

Rather than focusing solely on dashboard development, this project covers the complete analytics lifecycle—from synthetic data generation and data engineering to semantic modeling and executive reporting.

---

# 🎯 Business Problem

Operational data generated by airlines is often distributed across multiple systems and contains common data quality challenges such as:

- Missing or null values
- Duplicate records
- Inconsistent data types
- Invalid timestamps
- Incorrect delay values
- Data integrity issues across related datasets

Without proper validation and transformation, these issues lead to inaccurate KPIs, unreliable reports and poor business decisions.

This project addresses these challenges by implementing a scalable data engineering pipeline using Microsoft Fabric and Apache Spark before exposing trusted data through an executive Power BI dashboard.

---

# 🚀 Project Objectives

This project was designed with the following objectives:

- Generate realistic airline operational datasets using Python
- Build a scalable Lakehouse architecture using Microsoft Fabric
- Implement the Medallion Architecture (Bronze → Silver → Gold)
- Validate and cleanse raw datasets using Apache Spark
- Create optimized Delta Tables for analytical workloads
- Build a Warehouse and Semantic Model for enterprise reporting
- Develop interactive Power BI dashboards with advanced DAX measures
- Deliver meaningful operational insights for business users
